# PRD - 文件统一命名规范应用
## Forvis Mazars - 文件自动重命名系统

**版本：** 2.1
**日期：** 2026-03-18
**状态：** DEF（定稿）
**作者：** 开发团队

---

## 1. 执行摘要

本文档定义了一个文件自动重命名应用的产品需求，该应用实施 Forvis Mazars 的**文件统一命名规范**（FORMAZ-PRO-01 v2.0）。用户可以将文件拖拽到 Web 界面，通过引导式问答自动获取符合规范的文件名，命名结构如下：

```
<客户别名>-<AX服务代码>-<服务期间>-<文档缩写>-<文档日期>-<版本>-<文档状态>
```

即：
```
<ALIAS_CLIENTE>-<SERVICIO_AX>-<PERIODO_SERVICIO>-<ACRÓNIMO_DOCUMENTO>-<AÑO_FECHA_DOCUMENTO>-<VERSIÓN>-<ESTADO_DOCUMENTO>
```

此外，系统将通过聊天机器人与 **Microsoft Teams** 集成，用户可直接在企业通信平台中完成文件重命名。

---

## 2. 问题与机遇

### 2.1 当前问题
- 员工文件命名不一致，导致追溯和搜索困难
- 手动查询目录（缩写、AX 服务代码、状态）效率低且容易出错
- 缺乏在存储前自动验证命名规范的工具
- 命名不标准化阻碍了与自动化系统（RPA、AI、SharePoint）的集成

### 2.2 机遇
- 100% 自动化命名流程可大幅减少人为错误
- 与 Teams 集成可实现大规模推广，无需改变工作流程
- 自动验证合规性（ISO 9001、ISQM、ENS、ISO 27001）
- 为未来与 SharePoint、Power Automate 和 ERP AX 集成奠定基础

## 6. 功能需求

### 6.1 RF-01：拖拽界面

**描述：** 用户可以将一个或多个文件拖拽到 Web 应用的放置区域。

**验收标准：**
- 放置区域接受任何类型的文件
- 支持同时拖拽多个文件
- 放置后显示文件原始名称
- 自动检测并保留文件扩展名

- 自动转换为 YYYYMMDD 格式
- 用户可修改
- 必填项

### 6.7 RF-07：VERSIÓN 字段（版本 - 自动检测 + 手动）

**描述：** 文档的递增版本号（v1, v2, v3...）。

**验收标准：**
- 默认值：v1
- 如果拖入的文件名中已包含版本号（如 "informe-v2.docx"），自动检测并建议 v3
- 检测模式："v1"、"V1"、"v01"、"version1"、"ver1"、"_v1"、"-v1"
- 用户可手动修改
- 仅允许递增版本值：v1, v2, v3...
- 输出格式标准化：v1, v2, v3（小写，无前导零）

### 6.8 RF-08：ESTADO_DOCUMENTO 字段（文档状态 - 目录搜索）

**描述：** 文档生命周期状态。

**状态目录：**

| 状态 | 描述 |
|------|------|
| DRF | 草稿（Draft）。文档处于初始编写阶段，可能会有结构或内容上的修改。不可用于业务操作。 |
| REV | 审阅中。文档正在由指定负责人（IT、法律、合规、管理层等）进行验证。 |
| OBS | 有意见。文档已审阅但需在批准前进行调整，包含待解决的意见。 |
| APR | 已批准。文档已由指定负责人正式验证，待签署（如适用）。 |
| DEF | 最终版。最终批准并发布的版本，适用于实施。 |
| FDO | 已签署。文档已由管理层或授权负责人正式签署，具有官方效力。 |
| VIG | 生效中。文档目前在组织内适用。 |
| SUS | 暂停。文档暂时停止适用。 |
| OBSOL | 已过时。文档已正式撤回并被新版本替代。 |
| ARC | 已归档。无运营效力的历史文档，因法律或法规要求而保留。 |

**验收标准：**
- 带搜索功能的下拉菜单
- 选择时显示完整描述
- 默认值：DRF（草稿）
- 必填项
- **业务规则：** 状态为 DEF 或 FDO 的文档不能回退到 DRF

### 6.9 RF-09：预览与确认

**描述：** 重命名前显示最终名称预览。

**验收标准：**
- 并排显示当前名称与新名称
- 用不同颜色高亮名称中的每个组件
- 验证所有字段是否已填写
- "重命名"和"取消"按钮
- 确认名称符合所有规则（无空格、无特殊字符、YYYYMMDD 日期格式、连字符作为分隔符）
- 显示保留的原始扩展名

### 6.10 RF-10：批量重命名

**描述：** 允许拖拽多个文件并应用相同的基础配置。

**验收标准：**
- 支持同时拖拽多个文件
- 共享公共字段（客户、服务、期间）
- 文档缩写和版本按文件单独配置
- 确认前预览所有名称
- 显示重命名进度

### 6.11 RF-11：业务规则验证

**描述：** 应用 FORMAZ-PRO-01 程序的所有规则。

**待验证规则：**
1. 必须使用连字符（-）作为分隔符
2. 名称中不允许空格和特殊字符
3. 日期严格遵循 YYYYMMDD 格式
4. 版本必须递增（不能从 v1 跳到 v3 而没有 v2）
5. DEF 或 FDO 状态的文档不能回退到 DRF
6. 被替代的文档必须标记为 OBSOL
7. 仅允许官方目录中的缩写和服务代码
8. 客户别名不能包含特殊字符

### 6.12 RF-12：数据库验证与完整性检查

**描述：** 服务器端 SQLite 数据库系统，记录每个重命名文件的信息，支持完整性验证和版本建议。

**验收标准：**
- 文件成功重命名后，在客户端计算其 SHA-256 哈希值，连同生成的文件名和命名字段一起存储到数据库
- 当新文件在客户别名、服务代码、服务期间和文档缩写上与已有记录匹配（排除文档日期、版本和状态），系统自动建议下一个可用版本号
- 版本建议会显示相关文件作为参考，但用户仍可手动选择任意版本
- 上传已按命名规范命名的文件时，系统计算其哈希值并与数据库记录比对；如内容已变更，显示警报建议更改文件名或递增版本号
- 数据库存储在 `/home/rootadmin/data/nomenclatura/nomenclatura.db`，应用启动时自动创建
- 页脚显示数据库连接指示器（绿色 = 已连接，红色 = 不可用）
- 所有数据库功能均为可选 —— 应用在没有后端 API 时正常运行

**技术组件：**
- 后端：Express.js + better-sqlite3，端口 3001
- 哈希：Web Crypto API (SHA-256)，在浏览器中计算
- 代理：Nginx 将 `/api/*` 转发到 Node.js 后端
- 端点：`/api/health`、`/api/records`、`/api/records/match`、`/api/records/verify`

### 6.13 RF-13：历史记录与收藏

**描述：** 本地存储常用配置。

**验收标准：**
- 保存最近 50 次重命名记录
- 支持历史记录搜索
- 可将常用组合标记为"收藏"以便快速重用
- 支持导出/导入配置

## 12. 详细业务规则

### 12.1 命名规则（来自 FORMAZ-PRO-01 程序）

| # | 规则 | 验证方式 |
|---|------|----------|
| R1 | 必须使用连字符作为分隔符 | 正则：不允许空格、下划线或其他分隔符 |
| R2 | 不允许空格和特殊字符 | 正则：仅允许 [A-Za-z0-9-] |
| R3 | 日期格式必须为 YYYYMMDD | 日期验证 + 格式验证 |
| R4 | 初始版本始终为 v1 | 默认值 + 验证 |
| R5 | 版本变更必须递增 | v(n) -> v(n+1) |
| R6 | DEF 或 FDO 文档不能回退到 DRF | 状态转换验证 |
| R7 | 被替代的文档必须标记为 OBSOL | 用户警告 |
| R8 | 客户代码必须唯一 | 格式验证 |
| R9 | 缩写仅限官方目录 | 列表验证 |
| R10 | 文件扩展名始终保留 | 自动处理 |

### 12.2 推荐目录结构
```
/ 年份 / 客户 / 领域 / 服务 / 文档类型
```
示例：`/2026/FMZ00/Auditoría/AUD_CAOIN/Informes/`

### 12.3 允许的状态转换

```
DRF ──▶ REV ──▶ OBS ──▶ REV（审阅循环）
                  │
                  ▼
                APR ──▶ DEF ──▶ FDO ──▶ VIG
                                         │
                                    ┌────┴────┐
                                    ▼         ▼
                                   SUS      OBSOL ──▶ ARC
```

状态说明：
- DRF（草稿）→ REV（审阅中）→ OBS（有意见）→ 返回 REV 循环
- REV → APR（已批准）→ DEF（最终版）→ FDO（已签署）→ VIG（生效中）
- VIG 可转为 SUS（暂停）或 OBSOL（已过时）
- OBSOL → ARC（已归档）
